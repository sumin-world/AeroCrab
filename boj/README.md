# 📘 Baekjoon Algorithm Roadmap

> 백준 단계별 문제들을 효율적으로 재구성한 커리큘럼  
> 기초 → 자료구조 → 탐색 → DP → 심화 순

## 🎯 난이도 매핑
- **0-2단계:** 브론즈~실버 하위
- **3-5단계:** 실버~골드 중위
- **6-8단계:** 골드 상위~플래티넘 하위
- **9-12단계:** 플래티넘 중위~상위
- **13-16단계:** 다이아 하위~상위

## 📚 투 포인터 vs 슬라이딩 윈도우 개념 정리

### 🎯 투 포인터 (Two Pointers)
- **구간 길이가 가변적**
- 두 포인터가 조건에 따라 자유롭게 움직임
- 주로 합/차이가 특정 조건을 만족하는 구간 찾기

### 🪟 슬라이딩 윈도우 (Sliding Window)  
- **구간 길이가 고정적**
- 정해진 크기의 "창문"을 한 칸씩 이동
- 주로 고정 길이 구간에서 최대/최소값 찾기

## 💡 **문제별 핵심 패턴 정리**

### 🎨 패턴 1: 기본 투 포인터 (같은 방향)
```c
int left = 0, right = 0;
while (right < n) {
    // right를 포함한 구간 처리
    while (/* 조건 만족 시 */) {
        // left 이동
        left++;
    }
    right++;
}
```
**대표 문제:** 2003, 1806, 12892

### 🎨 패턴 2: 양방향 투 포인터 (양 끝에서 시작)
```c
int left = 0, right = n-1;  
while (left < right) {
    if (arr[left] + arr[right] == target) {
        // 정답 처리
    } else if (arr[left] + arr[right] < target) {
        left++;
    } else {
        right--;
    }
}
```
**대표 문제:** 3273, 2470, 2473

### 🎨 패턴 3: 슬라이딩 윈도우 (고정 크기)
```c
for (int i = 0; i < k; i++) sum += arr[i]; // 초기 윈도우
for (int i = k; i < n; i++) {
    sum += arr[i] - arr[i-k]; // 슬라이딩
    // 결과 갱신
}
```
**대표 문제:** 2531, 15961, 11003

### 🎨 패턴 4: Meet in the Middle + 투 포인터
```c
// 절반씩 나누어 모든 부분합 구하기
// 정렬 후 투 포인터로 조건 만족하는 쌍 찾기
```
**대표 문제:** 1208, 7453, 2143

- **O(N²) → O(N)**으로 시간복잡도 혁신적 개선
- **메모리 효율적** - 추가 공간 거의 불필요  
- **구현이 비교적 간단** - 복잡한 자료구조 불필요
- **응용 범위 넓음** - 다른 알고리즘과 결합 가능
